# Skill: PR Worker

## Goal
Turn validated findings into small, mergeable pull requests.

## Steps
1. Confirm issue scope and acceptance criteria.
2. Implement minimal diff.
3. Run required tests/lint/typecheck.
4. Write PR description with evidence.
5. Include risk + rollback note.

## PR Template Snippets
- Problem:
- Root cause:
- Fix:
- Validation:
- Risks:
- Rollback:

## Output
- `PR_CREATED` with PR URL
- or `PR_BLOCKED` with exact failing gate
